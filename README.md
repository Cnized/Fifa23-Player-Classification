# ⚽ Football Player Analysis & Classification Using Data Mining

Predicting player positions and discovering natural player archetypes from FIFA 23 and Transfermarkt data using classification and clustering techniques.

> Data Mining course project — Islamic Azad University, Central Tehran Branch (2nd Semester 1404–1405)

---

## 📋 Overview

This project applies the full data mining pipeline — from raw data collection to classification and clustering — on real football data. Two independent datasets (FIFA 23 player stats and Transfermarkt transfer records) are integrated, cleaned, and analyzed to:

1. **Predict a player's position** (Forward / Midfielder / Defender / Goalkeeper) from their performance attributes using supervised classification.
2. **Discover natural player archetypes** (e.g. "Elite All-Rounders", "Attacking Players") using unsupervised clustering — without using position labels.

## 🎯 Objective

Build a data mining system that takes a player's raw stats (pace, shooting, passing, dribbling, defending, physicality, overall, potential, age) and:
- Classifies their playing position with high accuracy
- Groups similar players together based on playing style alone

This mirrors real scouting workflows used by clubs like FC Barcelona and Manchester City.

## 📊 Datasets

| Dataset | Records | Columns | Source |
|---|---|---|---|
| **FIFA 23 Complete Player Dataset** | 18,539 players | 90 attributes | [LilianaC/Pandas](https://github.com/LilianaC/Pandas/blob/master/Fifa%2023%20Fut%20Players.csv) |
| **Transfermarkt Datasets** | 77,982 transfer records | 16 columns | [dcaribou/transfermarkt-datasets](https://github.com/dcaribou/transfermarkt-datasets) |

Transfermarkt data spans **4 European leagues** (La Liga, Premier League, Bundesliga, Serie A) from **1992–2025**.

## 🧪 Methodology

The project follows the standard KDD (Knowledge Discovery in Databases) pipeline:

```
Data Collection → Data Understanding → Preprocessing → Classification → Clustering → Evaluation
```

### Phase 1 — Data Understanding
- Attribute type analysis (numeric, categorical, boolean)
- Descriptive statistics (mean, median, std)
- Distribution visualization (histograms, correlation heatmap)

### Phase 2 — Preprocessing
- **Cleaning:** Missing value imputation (median/zero-fill), null-row removal
- **Integration:** Merging 4 league CSVs into a unified dataframe
- **Reduction:** Feature selection — 90 columns → 9 key performance attributes
- **Transformation:** Min-Max normalization to [0, 1], label encoding, transfer-type filtering

### Phase 3 — Classification
Four classifiers trained on an 80/20 stratified train-test split (14,831 / 3,708 records):

| Model | Accuracy |
|---|---|
| Decision Tree | 80.12% |
| Logistic Regression | 81.80% |
| K-Nearest Neighbors (K=121) | 84.57% |
| **Random Forest (100 trees)** | **86.89%** 🏆 |

**Best model:** Random Forest — `DefendingTotal` (22%) and `ShootingTotal` (18%) were the strongest predictors.

### Phase 4 — Clustering
K-Means clustering (K=4, validated via Elbow Method) revealed 4 natural player groups:

| Cluster | Players | Profile |
|---|---|---|
| Defensive Players | 5,460 | High defending & physicality |
| Elite All-Rounders | 5,651 | Best overall, balanced stats |
| Attacking Players | 5,367 | High pace, shooting, dribbling |
| Goalkeepers | 2,061 | Separated pre-clustering (unique stat space) |

> ⚠️ **Note:** Goalkeepers were manually separated *before* clustering — an initial K-Means run on all players incorrectly mixed GKs into outfield clusters due to their fundamentally different attribute space.
